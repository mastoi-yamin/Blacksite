# Contributing

Thank you for your interest in BLACKSITE.

This project follows strict engineering and design standards to maintain consistency and long-term maintainability.

---

## Before Contributing

Please read all documentation inside the `docs/` directory.

Understanding the project's architecture and design philosophy is required before contributing.

---

## Development Rules

* Respect the locked architecture.
* Do not rename folders.
* Do not reorganize files.
* Keep components focused.
* Follow the coding standards.
* Follow the design guidelines.
* Write readable TypeScript.
* Keep commits small and meaningful.

---

## Pull Requests

Every pull request should:

* Solve a single problem.
* Follow project conventions.
* Be reviewed before merging.
* Avoid unrelated changes.

---

## Code Quality

Contributions should prioritize:

* Readability
* Maintainability
* Accessibility
* Performance
* Simplicity

---

## Documentation

Whenever architecture or behavior changes, update the corresponding documentation.

Documentation is considered part of the product.

---

Thank you for helping maintain the quality of BLACKSITE.
