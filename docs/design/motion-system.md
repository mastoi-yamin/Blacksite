# Motion System

## Philosophy

Motion exists to guide the user, not distract them.

Every animation should have a purpose.

Animations should feel smooth, intentional, and premium.

Avoid excessive movement.

---

## Motion Principles

* Fast but not rushed.
* Smooth instead of flashy.
* Consistent across the application.
* Never block user interaction unless necessary.
* Respect reduced-motion preferences.

---

## Animation Style

Preferred:

* Fade
* Slide
* Scale
* Blur
* Opacity
* Subtle rotation

Avoid:

* Bounce
* Shake
* Rubber band
* Flash
* Excessive spinning

---

## Timing

Fast interactions:
100–200ms

Standard transitions:
250–400ms

Large page transitions:
500–800ms

Boot sequence:
Maximum 4 seconds.

---

## Scroll Animations

Reveal content only once.

Animations should support storytelling.

Avoid animating every element individually.

---

## Loading Experience

Loading should feel intentional.

Skeletons are preferred over spinners whenever possible.

Never leave the user wondering if something is broken.

---

## Golden Rule

If removing an animation improves the experience, the animation should not exist.
