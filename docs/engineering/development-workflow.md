# Development Workflow

## Philosophy

Development should be predictable, incremental and production-focused.

Avoid large unreviewed changes.

Every commit should improve the project.

---

## Workflow

1. Decide the feature.
2. Design the implementation.
3. Implement one file or one small feature.
4. Review the code.
5. Test the feature.
6. Commit the changes.

Repeat.

---

## AI Workflow

AI assists development.

AI does not make architectural decisions.

AI must never reorganize the project.

AI must only generate the files requested.

Every AI-generated file must be reviewed before merging.

---

## Git

Write clear commit messages.

Commit frequently.

Avoid unrelated changes in the same commit.

Keep commits small and meaningful.

---

## Code Reviews

Every feature should be reviewed before merging.

Check:

* Readability
* Maintainability
* Accessibility
* Performance
* Architecture compliance

---

## Feature Development

Never build multiple major features simultaneously.

Complete one feature before starting another.

---

## Refactoring

Refactor only when there is a clear benefit.

Avoid unnecessary rewrites.

Do not change working code without reason.

---

## Documentation

Keep documentation synchronized with implementation.

If architecture changes, update documentation immediately.

Documentation is part of the product.

---

## Final Principle

Build software that your future self will enjoy maintaining.
